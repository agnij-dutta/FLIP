# Contributing to FLIP

Thank you for your interest in contributing to FLIP!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/FLIP.git`
3. Set up development environment: `./scripts/setup-dev-env.sh`
4. Create a branch: `git checkout -b feature/your-feature-name`

## Code Style

- **Solidity**: Follow [Solidity Style Guide](https://docs.soliditylang.org/en/latest/style-guide.html)
- **Python**: Follow PEP 8, use black for formatting
- **Go**: Follow [Effective Go](https://go.dev/doc/effective_go) guidelines

## Testing

- Run tests before submitting PRs
- Ensure all tests pass: `npm test` and `forge test`
- Add tests for new features

## Pull Requests

1. Update documentation if needed
2. Add tests for new functionality
3. Ensure all CI checks pass
4. Write clear commit messages
5. Reference any related issues

## Security

If you discover a security vulnerability, please email security@flip-protocol.org instead of opening a public issue.
